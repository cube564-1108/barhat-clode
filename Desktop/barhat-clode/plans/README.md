# Plans — технические планы реализации

## Правила

1. **Один план = одна функция.** Не смешивать несколько задач в один файл.
2. **Имя файла:** `YYYY-MM-DD-название-функции.md`
3. **Фазы с чекбоксами** `[ ]` (не начата) / `[x]` (завершена)
4. **Агент обязан** обновлять план после каждой сессии.
5. **В конце плана** — итоговый блок: реализовано / частично / не начато.

## Шаблон плана

```markdown
# [Название функции] — план

**Статус:** [ ] В процессе / [x] Завершён  
**Дата:** YYYY-MM-DD  
**Цель:** [1-2 предложения что должно получиться]

---

## Фазы

### [ ] Фаза 1 — [Название]
- [ ] Шаг 1.1
- [ ] Шаг 1.2

### [ ] Фаза 2 — [Название]
- [ ] Шаг 2.1

---

## Итог

[Что реализовано, что осталось, что поменялось по ходу]
```

## Цикл создания плана (3 чата Миллера)

```
Чат 1: «Создай план реализации [функция]. Поищи как другие решали.»
        → черновой план → сохрани в plans/YYYY-MM-DD-название.md

Чат 2: «Прочитай план. Разбей на фазы с чекбоксами.»
        → план с фазами

Чат 3: «Проверь каждую фазу:
        - Лучшее ли решение?
        - Конфликты между фазами?
        - Безопасность?
        - Соответствие бизнес-целям?»
        → финальный план

Чат 4+: реализация фаза за фазой
```
